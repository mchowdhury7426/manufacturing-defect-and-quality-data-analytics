Quality Data Analytics — Closed‑Loop Quality Intelligence (RMG)

Mohaimin Chowdhury

Overview

This case study explains how I transformed inconsistent factory quality data into a scalable analytics system. It started as a simple requirement: identify top defects in factory endline. The real challenge was that real-world manufacturing data was not structured, not consistent, and not analysis-ready. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

The solution evolved into a complete quality intelligence build: cleaning and standardizing raw data, reshaping it into an analytics-friendly model, unifying multiple inspection stages, scaling the database ingestion approach, integrating buyer final inspection data, extending the model upstream to cutting/recutting/fabric, and finally closing the loop by mapping Voice of Guest (VoG) feedback into defect taxonomy and factory actions. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)[2](https://hoplun-my.sharepoint.com/personal/raju_ahmed_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7BD674EC8D-64BF-4200-8FAE-4A3C601C31B8%7D&file=VoG%20-%20Cloude%20Propmt.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

Case Study — Detailed Actions (Interpreted)

This section explains what I did, why I did it, and what each decision enabled. It is written from my original work notes in Quality Data. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

Challenge 1 — Data sourcing and cleaning

Context: The factory had a database, which is modern compared to many factories, but the data was stored without a practical analytics purpose. Input was partially manual and partially automatic, causing high error risk and inconsistency (typos, multiple date formats, buyer name mismatch, and mismatch in date and quantity). The dataset volume was large, with many historical Excel files. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

Actions:

  • Standardized dates: I converted multiple date formats into a single format to make time-based analysis reliable. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)
  • Corrected repeated errors systematically: I focused on repeated mistake patterns in style and product IDs and corrected them consistently across files. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)
  • Validated against official references: I checked IDs and naming against official lists to identify and correct mismatches. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)
  • Protected the operator workflow: I avoided changing the original entry structure because it would disrupt the data-entry operators and the ongoing database process. Instead, I planned to keep source structure and do the transformation inside Power BI. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

Interpretation: This stage was not “just cleaning.” It established trust in the data so that defect ranking would not mislead teams. It also ensured the solution could survive daily operations without forcing operators into a new process. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

Challenge 2 — Data structure in Power BI

Problem: Defect names existed as columns and defect counts were input in rows. There was no unique identifier for each record. In that structure, reliable “top defects” analysis is not possible. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

Actions:

  • Structured the dataset on load: I formatted each column and removed unnecessary columns to stabilize the model. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)
  • Created a unique record key: I added an index and built a unique ID by concatenating style, product ID, date, and index to uniquely identify each row. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)
  • Unpivoted defects for analytics: I duplicated the table and unpivoted defect columns so defects became rows with “Defect Name” and “Qty.” [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)
  • Built correct relationships: I linked the original and unpivoted tables using the unique ID (one-to-many), enabling correct slicing and ranking by line, floor, buyer, product, and date. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

Interpretation: The key insight here was: keep the source “operator-friendly,” but create an “analytics-friendly” structure in Power BI. That is what made accurate defect ranking possible without changing the factory’s input behavior. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

Challenge 3 — Putting it all together (Inline + Endline + Prefinal)

Problem: Inline and prefinal had different structures. For meaningful comparison and correlation across inspection phases, all stages needed to align with a common structure. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

Actions:

  • Added an “inspection stage” identifier to each dataset to clearly separate Inline, Endline, and Prefinal. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)
  • Aligned structures and appended them into a single “central database” table. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)
  • Created a central unpivoted defect table from the central database for defect ranking across all stages. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)
  • Connected tables deliberately so filtering works stage-wise and defect-wise (central as the base, unpivot as the defect fact). [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

Interpretation: This is where the solution moved from a single report to a system. It enabled stage comparison (Inline vs Endline vs Prefinal) and created a stable foundation for multi-stage RCA. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

Challenge 4 — Database scalability (Excel limitations)

Problem: Data was in Excel and the volume was too large. A single-file database would not scale, but moving to a complex system could overwhelm entry operators. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

Actions:

  • Adopted folder-based storage: Instead of storing everything in one file, the database was split into smaller standardized Excel files. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)
  • Connected Power BI to folders: Power BI reads the folder so dropping new files updates the dataset without manual rebuild. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)
  • Improved entry templates: Files were formatted with lookups and error-return logic so mistakes are flagged early in the entry stage. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

Interpretation: This was the “scale without stress” move—keeping Excel for human usability while making Power BI behave like a scalable database ingestion system. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

Challenge 5 — Integrating final data from buyer

Problem: Internal final data lacked reliability, so buyer final reports were preferred. But buyer reports had different defect naming and different report structures, including grouped categories (measurement, workmanship, packaging). [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

Actions:

  • Transformed buyer reports using Power Query to restructure them into a usable analytic format. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)
  • Built DAX-based calculations and designed a model for buyer final data before integrating it with factory inspection stages. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)
  • Handled grouped categories through filtering and logic so analysis remains meaningful and segmented. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)
  • Enabled structured problem-solving (5-Why and PDCA) using buyer findings as triggers for factory-side investigation. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

Interpretation: This step made the system buyer-aligned. Instead of only internal reporting, it supported investigations starting from buyer final results and tracing back into factory stages. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

Challenge 6 — Integration of cutting, recutting, and fabric data

Problem: Cutting/recutting integration required consistent references (style + PID + item type). Fabric data did not map directly to style because it was contract-based. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

Actions:

  • Created a style reference table mapping style with PID and item type so the overall model can be based on style. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)
  • Created a fabric reference table to connect fabric using contract numbers. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)
  • Integrated stages into one system so issues can be investigated across upstream and downstream steps. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

Challenge 8 — Integration of Voice of Guest (VoG)

Problem: Stakeholders wanted customer review analysis to identify defects, but manual mapping was impossible at scale. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

Actions:

  • Used an AI-assisted interpretation workflow (Copilot agent + Claude mentioned in your notes) to map comments into defect type, category, and name. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)
  • Applied disciplined classification rules from your VoG prompt design: sentiment filled for every row, and for negative rows one dominant issue is assigned using your defect list whenever possible. [2](https://hoplun-my.sharepoint.com/personal/raju_ahmed_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7BD674EC8D-64BF-4200-8FAE-4A3C601C31B8%7D&file=VoG%20-%20Cloude%20Propmt.docx&action=default&mobileredirect=true&DefaultItemOpen=1)
  • Integrated VoG into the same Power BI ecosystem so a customer complaint can trigger factory-level investigation from fabric to cutting to sewing to delivery. [1](https://hoplun-my.sharepoint.com/personal/mohaimin_chowdhury_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7B465F1D04-D319-4988-9BEE-495B8A0BB1FE%7D&file=Quality%20Data.docx&action=default&mobileredirect=true&DefaultItemOpen=1)[2](https://hoplun-my.sharepoint.com/personal/raju_ahmed_hoplun_com/_layouts/15/Doc.aspx?sourcedoc=%7BD674EC8D-64BF-4200-8FAE-4A3C601C31B8%7D&file=VoG%20-%20Cloude%20Propmt.docx&action=default&mobileredirect=true&DefaultItemOpen=1)

Closing Reflection

This started as “find the top defects,” but the core work was building a system: reliable inputs, analytics-ready structure, stage integration, scalable ingestion, buyer-aligned final insights, upstream connection, and customer-feedback closure. The outcome is a single analytics environment designed to support faster, clearer quality decision-making.
